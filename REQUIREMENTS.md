# 书虫书店 — 功能需求文档

## 1. 现有功能概览

| 功能模块 | 描述 |
|---------|------|
| 商品展示 | 网格布局展示书籍卡片（封面Emoji、书名、作者、价格、库存、分类标签） |
| 分类筛选 | 按文学/小说/科幻/科技/儿童/历史/社科 7 个分类切换筛选 |
| 关键词搜索 | 按书名或作者进行模糊搜索（防抖300ms） |
| 价格区间筛选 | 输入最低价/最高价进行价格范围过滤 |
| 排序功能 | 默认/价格从低到高/价格从高到低/书名A-Z/库存从多到少 |
| 购物车 | 侧边栏购物车，支持增减数量、删除、清空 |
| 本地存储 | 购物车数据和库存数据持久化到 localStorage |
| 结算下单 | 表单验证（姓名/手机号/地址），提交后扣减库存 |
| 订单成功弹窗 | 下单成功后展示订单详情 |
| Toast 通知 | 操作反馈提示 |
| 响应式布局 | 适配桌面/平板/手机 |
| 快捷键 | Ctrl+F 展开搜索栏 |

## 2. 新增功能设计

### 2.1 收藏夹 / 心愿单

#### 功能描述
用户可以将心仪的书籍添加到收藏夹（心愿单），方便后续快速找到并购买。收藏夹独立于购物车，书籍即使缺货也可收藏。

#### 交互流程
1. 用户在商品卡片上点击"♡ 收藏"按钮（心形图标）
2. 按钮切换为"❤️ 已收藏"状态，同时出现 Toast 提示"已加入收藏"
3. 再次点击取消收藏，Toast 提示"已取消收藏"
4. 用户可通过导航栏"❤️ 收藏夹"按钮打开收藏夹侧边栏
5. 收藏夹侧边栏展示所有已收藏书籍，每项可"加入购物车"或"取消收藏"
6. 收藏夹中的书籍如果已加入购物车，显示"已在购物车"标识

#### 数据结构
```js
{
  wishlist: [
    { id: 1, addedAt: "2026-05-19T10:30:00Z" }
  ]
}
```

#### 数据存储
- localStorage Key: `book_wishlist`
- 仅存储 book id 和收藏时间，书籍详情实时从 PRODUCTS 读取

#### 验收标准
- [ ] 点击收藏按钮可切换收藏/取消收藏状态
- [ ] 收藏的书籍在刷新页面后仍保持收藏状态
- [ ] 收藏夹侧边栏可正确展示所有收藏书籍
- [ ] 从收藏夹可直接将书籍加入购物车
- [ ] 缺货书籍也可收藏
- [ ] 收藏夹按钮显示收藏总数角标
- [ ] 响应式适配移动端

### 2.2 书籍详情弹窗

#### 功能描述
点击商品卡片进入书籍详情弹窗，展示更丰富的书籍信息，包括书籍描述、出版信息、用户评分等。

#### 交互流程
1. 用户点击商品卡片（除按钮区域外）
2. 弹出书籍详情弹窗（modal），展示完整信息
3. 弹窗内容包括：大号封面Emoji、书名、作者、分类标签、价格、库存状态
4. 扩展信息：书籍简介/描述、出版社、出版年份、ISBN、页数
5. 弹窗底部提供"加入购物车"和"❤收藏"按钮
6. 点击弹窗外遮罩或关闭按钮关闭弹窗

#### 数据结构 — PRODUCTS 扩展
```js
{
  id: 1,
  name: '红楼梦',
  author: '曹雪芹',
  category: '文学',
  price: 59.00,
  stock: 30,
  emoji: '📕',
  description: '中国古典四大名著之一，以贾宝玉与林黛玉、薛宝钗的爱情婚姻悲剧为主线...',
  publisher: '人民文学出版社',
  publishYear: 1996,
  isbn: '9787020002207',
  pages: 1600,
  rating: 4.8,
  ratingCount: 12530
}
```

#### 验收标准
- [ ] 点击商品卡片可打开详情弹窗
- [ ] 弹窗展示完整扩展信息
- [ ] 弹窗中"加入购物车"按钮正常联动购物车
- [ ] 弹窗中收藏按钮正常联动收藏夹
- [ ] 弹窗可点击遮罩或关闭按钮关闭
- [ ] 响应式适配（小屏全屏展示）

### 2.3 用户评分与评价

#### 功能描述
用户可以对购买的书籍进行评分（1-5星）和撰写文字评价，帮助其他用户选书。

#### 交互流程
1. **下单成功后**：在成功弹窗中增加"去评价"入口
2. **评价入口**：在商品卡片底部展示星级评分（只读）
3. **写评价**：点击"写评价"打开评价表单弹窗
4. **评价表单**：选择星级（1-5星，点击交互）+ 文字评价（选填，最多200字）
5. **提交评价**：提交后更新该书的平均评分和评价数量
6. **评价展示**：在书籍详情弹窗中展示最近评价列表

#### 数据结构
```js
// 评价数据
{
  reviews: [
    {
      id: 1,
      bookId: 1,
      rating: 5,        // 1-5 星
      content: "非常经典的作品，值得反复阅读！",
      userName: "书虫用户",
      createdAt: "2026-05-19T10:30:00Z"
    }
  ]
}
```

#### 数据存储
- localStorage Key: `book_reviews`
- 评价与书籍关联，支持按 bookId 查询

#### 验收标准
- [ ] 商品卡片可展示星级评分（已评分书籍）
- [ ] 用户可对书籍进行1-5星评分
- [ ] 用户可撰写文字评价（最多200字）
- [ ] 书籍详情弹窗展示评价列表
- [ ] 平均评分根据所有评价动态计算
- [ ] 刷新后评价数据不丢失

### 2.4 订单历史

#### 功能描述
用户可以查看历史订单记录，包括订单编号、商品明细、总金额、下单时间、订单状态。

#### 交互流程
1. 在导航栏新增"📋 我的订单"按钮
2. 点击打开订单历史页面/侧边栏
3. 按时间倒序展示所有历史订单
4. 每个订单展示：订单编号、下单时间、商品列表（封面+名称+数量+小计）、总金额、订单状态
5. 支持清空历史订单

#### 数据结构
```js
{
  orders: [
    {
      orderNo: "BK1716000000000",
      createdAt: "2026-05-19T10:30:00Z",
      items: [
        { id: 1, name: '红楼梦', author: '曹雪芹', emoji: '📕', quantity: 2, price: 59.00 }
      ],
      total: 118.00,
      status: "completed"   // completed | cancelled
    }
  ]
}
```

#### 数据存储
- localStorage Key: `book_orders`
- 下单成功时自动保存订单到历史记录

#### 验收标准
- [ ] 下单成功后订单自动保存到历史
- [ ] 订单历史页面展示所有历史订单
- [ ] 订单按时间倒序排列
- [ ] 点击订单项可展开查看商品明细
- [ ] 支持清空所有历史订单
- [ ] 刷新后订单数据不丢失
- [ ] 订单编号格式统一（BK + 时间戳）

### 2.5 后台管理面板

#### 功能描述
提供一个管理员入口，用于管理书籍商品信息。管理员可以查看、添加、编辑和删除书籍。

#### 交互流程
1. **入口**：在页面底部或导航栏设置"管理后台"入口（可设计为隐藏入口，如连续点击Logo 3次）
2. **认证**：简单的密码验证弹窗（默认密码 `admin123`）
3. **管理面板**：以表格形式展示所有书籍，包含：ID、书名、作者、分类、价格、库存、操作按钮
4. **添加书籍**：表单输入完整书籍信息（含扩展字段），保存后刷新列表
5. **编辑书籍**：弹出已填充数据的编辑表单，修改后保存
6. **删除书籍**：确认弹窗后删除，同时清理相关的收藏和评价数据
7. **库存管理**：可直接修改库存数量

#### 数据结构
使用现有 PRODUCTS 数据结构，扩展上述 description/publisher/isbn 等字段。

#### 数据存储
- 基础数据修改同步到 localStorage `book_products`
- 如果没有 localStorage 数据，默认加载内置 PRODUCTS

#### 验收标准
- [ ] 管理员入口可触发登录验证
- [ ] 正确密码后进入管理面板
- [ ] 管理面板以表格展示所有书籍
- [ ] 支持添加新书籍（包含所有字段）
- [ ] 支持编辑已有书籍
- [ ] 支持删除书籍（含确认弹窗）
- [ ] 支持修改库存
- [ ] 修改后前端商品展示同步更新

## 3. 数据关系总览

```
localStorage:
  ├── book_stock      → { id: stock }                   // 库存数据
  ├── book_cart       → [{ id, name, author, ... }]     // 购物车
  ├── book_wishlist   → [{ id, addedAt }]                // 收藏夹 ← 新增
  ├── book_reviews    → [{ id, bookId, rating, ... }]    // 评价 ← 新增
  ├── book_orders     → [{ orderNo, items, total, ... }] // 订单历史 ← 新增
  └── book_products   → [{ id, name, author, ... }]      // 商品管理 ← 新增
```

## 4. 技术约束

- 纯前端实现（HTML + CSS + JavaScript），不引入外部框架/库
- 数据持久化使用 localStorage
- 所有UI风格与现有系统一致（暖色调、衬线字体、圆角卡片）
- 响应式适配现有断点（768px, 480px, 992px）
- 遵守现有代码规范（IIFE 模块化、window._book 暴露接口）
